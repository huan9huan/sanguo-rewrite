# v0.3.3 Release Notes

发布日期：2026-04-11

## 一句话

v0.3.3 把《三国演义》内容生产链路从“很多散乱文件和临时操作”收束成以 `current/` 为核心的 reading workflow，并让 CP002 正式进入网站阅读体验。

这一版的重点不是增加复杂功能，而是让内容生产、漫画改编、网站阅读和发布运维变得更清楚、更稳、更容易继续扩展。

## 主要变化

### 1. Reading workflow 收紧为 current-only

网站现在只消费 passage 的 `current/` handoff 文件。

核心约定：

- 正文入口：`current/draft_cn.md` 或 `current/approved_cn.md`
- 漫画入口：`current/comic.png` + `current/comic.json`
- 网站导出只从稳定 handoff asset 组装 reading model

这样做的结果是：网站不再需要理解 draft/run/version 历史，也不会在多个候选文件之间猜测应该读哪个。

### 2. CP002 接入阅读链路

CP002 已开始进入网站内容系统。

本轮新增和整理了：

- `cp002-p01`：破法斩将
- `cp002-p02`：孙坚登场
- `cp002-p03`：怒鞭督邮
- `cp002-p04`：忠臣死谏
- `cp002-p05`：引狼入室

章节列表和 passage 页面现在可以展示 CP002 内容，comic 资产也能进入 reading flow。

### 3. 漫画阅读体验优化

漫画页从“整张图被切开后强行放大”改成更适合阅读的流式体验。

本轮解决了几个关键问题：

- 中间窄竖格不再硬拉大，而是使用侧旁白布局。
- 横向 frame 保持原有比例，不被侧旁白规则破坏。
- mobile comic reading 去掉重边框和强卡片感，和 Text Reading 的流式布局保持一致。
- 旁白区域加宽，mobile 字体略小，减少无意义断行和截断。
- navigation header 在 scroll 顶部不再因为高度切换而颤抖。

### 4. Workflow Spec V1 更明确

本轮把 content AI workflow 进一步写清楚，形成更稳定的角色分工。

新增或强化的角色包括：

- `章节策划`：负责 chapter 拆 passage。
- `段落策划`：负责 passage spec、scene spec、短标题和单行 catchup。
- `正文写手`：根据 spec 写中文正文。
- `正文审稿`：判断正文是否可读、是否稳定。
- `正文改稿`：根据 review 修正文。
- `角色定妆`：漫画改编前检查新核心人物，维护 `memory/character_visuals.json`。
- `漫画改编`：把 current text 改成 comic prompt / layout contract。
- `漫画质检`：判断生成图是否可用。
- `阅读编排`：把 comic frame 插入正文阅读流。
- `工作区运维`：负责 promote、box detect、ImageMagick 图片转换。
- `发布运维`：负责本地导出验证、GCS 同步和部署。

`AGENTS.md` 现在也增加了常见任务路由示例，方便 AI 看到一句任务后快速定位该使用哪个 role。

### 5. Workspace Operator 更实用

工作区运维流程现在更明确：

- draft promote 进入 `current/`
- comic run 先 refresh boxes，再 promote
- `comic.png` 和 `comic.json` 必须一起进入 `current/`
- JPG/JPEG/WebP 输入会通过 ImageMagick 转成 web-friendly PNG
- 不再把 `published/` 当作当前 workflow 的一等目录

这让手工生成图片后的 promote 流程更直接，也减少了网站读取端的负担。

### 6. Release Operator 增加远程发布职责

发布运维现在不仅负责本地 `content:export`，还明确负责完整发布链路：

1. 本地导出 content。
2. 本地 build 验证。
3. 同步 `site/public/content/` 到远程 GCS。
4. 部署网站。
5. 确认 production 能读到最新 content。

典型命令：

```bash
cd site
npm run content:export
npm run build
gsutil -m rsync -r -d site/public/content gs://<content-bucket>/<prefix>
npm run deploy
```

## 网站体验变化

读者侧能感受到的变化：

- 章节/段落列表更清楚，有短标题和单行 catchup。
- CP002 已开始出现在阅读目录中。
- 漫画阅读在手机上更像正文流，不再像嵌在一堆重卡片里。
- 小漫画格不会被过度放大到模糊。
- header 滚动体验更稳定。

## 内部生产变化

创作者和 AI agent 侧能感受到的变化：

- 每个 agent 的责任更清楚，不需要猜“这个任务谁来做”。
- `current/` 是唯一网站 handoff surface。
- 漫画改编前要先确认核心人物视觉 canon。
- comic promote 产物固定为 `comic.png` + `comic.json`。
- 发布前有明确的本地验证、GCS 同步、部署顺序。

## 已知注意事项

- 当前 build 仍会出现一个 Turbopack NFT warning，已知但不阻断构建。
- 漫画 frame 仍来自整页图裁切。小图体验已通过侧旁白缓解；长期更优方案是导出独立 frame assets。
- GCS bucket / prefix 仍需要按实际部署环境填写。
