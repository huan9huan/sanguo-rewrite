# 三国演义 AI 插画生产系统设计文档 v3

## 1. 项目目标

做的不是单张 AI 画图，而是一套可以持续扩产的《三国演义》插画系统。

目标拆成 4 件事：

1. 场景结构化
2. 人物统一化
3. 生成流程自动化
4. 资产沉淀可复用

---

## 2. 设计原则

### 2.1 数据驱动
不要手工一张张写 prompt。场景、角色、风格都进 JSON。

### 2.2 一致性优先
系统里最贵的不是单张图的钱，而是人物和风格乱掉之后，整套项目不可用。

### 2.3 先跑通，再升级
先用文生图批量出图。后面再接参考图、图生图、LoRA。

---

## 3. 系统架构

```text
Scene DB
  ↓
Character DB
  ↓
Prompt Builder
  ↓
Engine Router
  ↓
Provider Adapter (Seedream / NanoBanana / Qwen)
  ↓
Candidate Pool
  ↓
Scorer / Human Review
  ↓
Approved Assets
```

## 4. 入场讨论与技术路线

### 4.1 为什么不是只选一个画图工具
你的需求不是“偶尔画一张”。而是：

- 有很多三国场景
- 要连载感
- 要角色统一
- 要后续可改图和复用

所以真正要选的是一套生产线，而不是一个单点工具。

### 4.2 为什么当前路线选 Ark + Seedream
原因：

- 可通过 API 自动化
- 单张成本可控
- 后续能接图生图、参考图
- 适合批量、适合程序化管理

### 4.4 为什么现在要升级为多引擎框架
原因：

- 后续要比较不同模型的单张成本和稳定性
- 同一套 prompt / 数据资产应该能复用到多个引擎
- 便于做“低成本批量初稿 + 高质量重点 refine”的路由策略
- 避免主流程和某一家 API 返回格式强耦合

当前已接入：

- Seedream
- Qwen Image

当前预留：

- NanoBanana

模型管理原则：

- Model 与 engine 绑定
- 运行时只选择 engine，不在 CLI 中临时切换 model
- 某个 engine 使用的具体 model 由环境变量或引擎配置统一管理

### 4.3 阶段策略

#### Phase 1
- scenes.json 驱动场景
- prompt 模板统一生成
- Seedream 文生图出 3 张候选
- 人工挑图

#### Phase 2
- 给核心角色制作标准参考图
- 场景图开始挂参考图
- 用图生图 refine

#### Phase 3
- 建评分器
- 做自动筛图
- 对关键角色训练 LoRA

## 5. 数据设计

### 5.1 scenes.json
这是场景宇宙。每个场景至少有：

- 标题
- 人物列表
- 地点
- 时间
- 情绪
- 动作
- 镜头
- 构图

### 5.2 characters.json
这是角色圣经。至少有：

- 视觉锚点
- 气质锚点
- 色板
- 标志物
- 禁止项

### 5.3 character_refs.json
这是角色标准像索引。后续角色一致性靠它。

---

## 6. Prompt 系统

Prompt 不只是描述场景，而是同时锁 4 层：

1. 场景叙事
2. 人物外观
3. 镜头语言
4. 禁止项

推荐结构：

- 标题
- 人物设定
- 场景信息
- 风格统一要求
- 避免项

## 8. 生成流程

### 标准流程

```text
读取场景
  ↓
拼 prompt
  ↓
生成候选图（默认3张）
  ↓
保存 metadata
  ↓
评分 / 人工挑图
```

### refine 流程（下一步）

```text
初稿
  ↓
图生图强化角色与风格
  ↓
成品入库
```

---

## 9. 质量控制

每张图至少看 4 个维度：

1. 角色一致性
2. 场景可读性
3. 构图质量
4. 风格统一性

当前仓库里先放占位评分器，后面再升级成自动多模态评分。

---

## 10. 成本与产能思路

推荐策略：

- 普通场景：3 张初稿里挑 1 张
- 关键场景：3 张初稿 + 1~2 次 refine
- 核心封面：单独高质量流程

这样可以把大多数图控制在合理成本里，同时把关键图拉高质量。

多引擎阶段建议额外记录：

- engine
- model
- latency
- estimated_cost
- candidate_count
- human_pick_rate

## 11. 当前 repo 可以做什么

- 改 characters.json，逐步把人物锁稳
- dry-run 查看 prompt

- 输出 metadata 和 scores
